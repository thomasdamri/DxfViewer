Part 1 — UI Audit 14 findings across 4 panels
Consistency
Defect List

Search placeholder
Wrong placeholder text. Reads "Search fittings…" — this is a defect panel. The History panel correctly says "Search defects…". Causes momentary confusion about what is being searched.
Defect List

LIVE status badge
LIVE renders as plain text, not a pill. SCHEDULED and CLOSED are correctly pill-shaped with tinted backgrounds. LIVE appears as raw right-aligned red text, breaking the badge system. All three statuses must share the same pill component.
Both defect panels

Classification labels
Classification labels are unstyled plain text. "Essential", "Routine", "Deferred" appear in the default text colour with no badge. Per spec they must be coloured mono badges using --cl-* variables — coloured text on a tinted background. This is the primary visual triage signal; its absence makes all cards look identical.
Both defect panels

Card left border
3 px classification border absent from all cards. Per spec the left border in --cl-*-mid is the second visual signal of classification category. Without it, combined with unstyled labels, the entire triage colour system is non-functional.
Defect History

Result count position
Count "6 defects" floats right-aligned on the toggle row. In the Defect List panel, "4 results" occupies its own line above the toolbar. These must be consistent — same position, same format, same font.
Defect History

Filter control pattern
Filter controls differ from the Defect List panel. History shows "Last 12 months ▼" and "Priority ↓" as separate buttons. The Defect List uses a unified "Filters ▼" button. Same content type, two different filter patterns.
Fittings List

Item tag numbers
Item numbers (001, 002…) rendered in the default sans font. These are identifiers and should use IBM Plex Mono — consistent with how all IDs are treated across the system.
Typography
Fitting Detail

Section labels
"Identification" and "Lifecycle" labels violate spec. They render in blue, proportional font — styled as interactive section headings. Per CLAUDE.md they must be: IBM Plex Mono, 10 px, uppercase, letter-spaced, tertiary muted colour. The blue implies interactivity where none exists.
Both defect panels

Classification label font
Classification text not monospaced. "Essential / Routine / Deferred" in the card header row uses proportional sans. Per spec all classification labels should use IBM Plex Mono.
Information Hierarchy
Both defect panels

Triage colour system
Classification colour system is entirely invisible. The left border + coloured badge together are the primary differentiator between triage categories. Both are absent. A user cannot distinguish an Essential LIVE defect from a Deferred CLOSED one at a glance.
Defect List

Keyword filter row
Active keyword chips cause a layout shift. When filters are applied, a new "Keywords: …" row appears below the toolbar, pushing all card content down. Prefer a stable row that is always present — status chips always occupy it, keyword chips join when active.
Filter & Search UX
Defect List

No "clear all" control
No bulk filter clear action. Each chip has an individual ×, but there is no way to clear all active filters at once. A "Clear filters" link should appear in the chip row when any keyword filter is active.
Missing & Unclear
Defect List

Back navigation
No way to return to the Fittings list. The Fitting Detail panel has a ← back arrow — this pattern should be consistent. The Defect List panel, shown in isolation, has no exit path.
Defect History

Entry point unclear
No "History" tab visible in the Fitting Detail panel. The detail shows General / Defects / Other. It is not clear how the Defect History view is reached. If it is a sub-view of "Other", this should be communicated.
